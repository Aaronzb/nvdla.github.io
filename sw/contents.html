Software Manual¶

NVDLA has a full software ecosystem including support from compiling network to inference. Part of this ecosystem includes the on-device software stack, a part of the NVDLA open source release; additionally, NVIDIA will provide a full training infrastructure to build new models that incorporate Deep Learning, and to convert existing models to a form that is usable by NVDLA software. In general, the software associated with NVDLA is grouped into two parts: the Compilation tools (model conversion), and the Runtime environment (run-time software to load and execute compiled neural network on NVDLA). The general flow of this is as shown in the figure below;

  • Compilation tools
    • Parser
    • Compiler
  • Runtime environment
    • User Mode Driver
    • Kernel Mode Driver
  • Test applications
    • Runtime test application
    • Compiler test application

Browsing Source Code¶

  • top (https://github.com/nvdla/sw)
    • umd: User Mode Driver
      • core: NVDLA specific implementation of user mode components
        • runtime: Runtime environment
        • compiler: Compilation tools
        • include: Header file for common implementation
        • common: Implementation shared between runtime and compiler such as loadable and logging
      • external: External modules used in UMD such as flatbuffers
      • include: Application Programming Interface
      • make: Make files
      • port: Portability layer
        • linux: Portability layer for Linux
      • tests: Test applications
        • runtime Runtime test application
        • compiler Compiler test application
    • kmd: Kernel Mode Driver
      • Documentation: Device tree bindings for NVDLA device
      • firmware: Core DLA hardware programming including HW layer scheduler
      • include: Core Engine Interface
      • port: Portability layer
        • linux: Portability layer for Linux
    • prebuilt: Prebuilt binaries
    • regression: Test regression framework
      • flatbufs: Pre-generated loadables for sanity tests
      • golden: Golden results
      • scripts: Scripts used for test execution
      • testplan: Test plans
    • scripts: General scripts
